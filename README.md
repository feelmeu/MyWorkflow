# MyWorkflow

AI Agent 任务执行工作流——一套让大模型在动手前先想清楚、在已有基础上搭建、对不确定的事保持诚实的方法论。

## 这是什么

一份 `workflow.md`，告诉 AI agent 在接到新任务时该怎么走：

1. **Grill** — 先问清楚再动手，不猜用户意图
2. **Issue Tracker** — 把计划写下来，可追溯、可交接
3. **ADR** — 重大决策有记录，未来不用重新推理
4. **外部参考** — 能搜先搜，不能搜标可信度
5. **状态管理** — 任务流转有章法，阻塞有超时
6. **Review** — 验收标准客观化，不靠"感觉差不多"

## 核心信条

- **省 token 不是少说话，是不走弯路**
- **减少幻觉不是变聪明，是诚实面对不确定性**
- **在已有基础上搭建，不凭空出现**

## 目录结构

```
├── workflow.md                  ← 核心工作流（给 AI 读的）
├── CONTEXT.md                   ← 项目术语表模板
├── .scratch/
│   ├── templates/
│   │   ├── 0001-issue-template.md   ← issue 模板
│   │   └── ADR-FORMAT.md            ← ADR 模板
│   └── issues/                      ← issue 实例
├── docs/
│   └── adr/                         ← ADR 实例
└── tech/
    └── python.md                    ← 技术栈术语增补示例
```

## 适用范围

通用，不绑定任何技术栈。issue 管理、术语表、ADR 目录等都可按项目实际情况替换。

## 使用方式

1. 将本仓库作为项目的工作流参考
2. 根据项目需要调整 `CONTEXT.md` 和 `tech/*.md`
3. 按 `workflow.md` 的流程引导 agent 执行任务
